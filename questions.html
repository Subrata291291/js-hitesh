What is memory management and garbage collection in JavaScript? **Answer:** JavaScript uses **automatic garbage collection** — the engine automatically frees memory that is no longer reachable. The main algorithm is **Mark and Sweep**: 1. GC starts from "roots" (global scope, current call stack) 2. It marks all objects reachable from roots 3. Global variables: // ❌ Accidental global (without let/const/var): function fn() { leakedVar = "oops"; // creates global variable! } // ✅ Always declare variables with const/let // 5. When would you use them? **Answer:** Web Workers run JavaScript in a **background thread**, separate from the main thread. This prevents heavy computation from blocking the UI. ```js // main.js — main thread const worker = new Worker("worker.js"); // Send data to worker: worker.postMessage({ data: [1, 2, 3, ...largeArray], operation: "sort" }); // Receive results from worker: worker.onmessage = function(event) { const sorted = event.data; renderResults(sorted); // update UI on main thread }; worker.onerror = (err) => console.error("Worker error:", err); // Terminate worker when done: worker.terminate(); // worker.js — background thread self.onmessage = function(event) { const { data, operation } = event.data; // Heavy computation that would normally freeze the UI: if (operation === "sort") { const sorted = data.sort((a, b) => a - b); self.postMessage(sorted); // send back to main thread } }; // Use cases for Web Workers: // ✅ Sorting/filtering huge datasets // ✅ Image/video processing // ✅ Cryptographic operations // ✅ Real-time data processing (stocks, analytics) // ✅ Complex calculations (physics simulations, ML inference) ``` --- ## Q107.
